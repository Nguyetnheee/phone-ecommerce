# GENERAL CODING SKILL

Before coding:
1. Read the current folder structure.
2. Identify the exact feature requested.
3. List files that need to be created or modified.
4. Ask for clarification if the request is ambiguous.

Coding rules:
- One feature per task.
- Keep code modular.
- Use clear names.
- No hardcoded business logic in UI components.
- No unused imports.
- No console.log in final code unless for temporary debugging.
- No fake security.
- No payment gateway integration unless requested.

Output format:
1. Summary of what was implemented.
2. Files changed.
3. How to run/test.
4. Notes or limitations.
